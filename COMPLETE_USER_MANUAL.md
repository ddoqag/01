## 系统概述

### 🎯 工具集架构

您的系统采用模块化设计，包含以下核心组件：

```
🏗️ 系统架构
├── 🤖 Claude Code Core (核心引擎)
├── 🔧 开发工具套件 (Development Tools)
│   ├── feature-dev (功能开发工作流)
│   ├── code-review (代码审查系统)
│   └── mcp-tools (项目健康分析)
├── 📊 分析工具套件 (Analytics Tools)
│   ├── MCP服务器集成 (MCP Integration)
│   └── DeepSeek AI分析 (DeepSeek Analytics)
├── 🤝 协作工具套件 (Collaboration Tools)
│   ├── commit-commands (Git自动化)
│   └── pr-review-toolkit (PR审查工具)
└── 🎨 智能代理系统 (AI Agents)
    ├── code-architect (代码架构师)
    ├── code-explorer (代码探索者)
    └── specialized-agents (专业代理)
```

### 🌟 核心优势

1. **智能化**: 基于Claude AI的智能决策支持
2. **集成化**: 无缝整合多种开发工具
3. **自动化**: 减少重复性工作，提高效率
4. **专业化**: 针对不同场景的专门化工具
5. **可扩展**: 支持自定义插件和工作流

## 安装与配置

### ✅ 系统要求

- **操作系统**: Windows 10+, macOS 10.15+, Linux
- **Python**: 3.8-3.11 (推荐 3.10)
- **Node.js**: 16.0+
- **Git**: 2.20+
- **Claude Code**: 最新版本

### 🔧 环境检查

检查您的系统是否已正确配置：

```bash
# 检查Claude Code
claude --version

# 检查Python
python --version

# 检查Node.js
node --version

# 检查Git
git --version
```

### ⚙️ 初始配置

#### 1. Claude Code配置
```json
// ~/.claude/settings.json
{
  "plugins": [
    "agent-sdk-dev",
    "commit-commands",
    "code-review",
    "feature-dev",
    "pr-review-toolkit"
  ],
  "mcpServers": {
    "deepseek": {
      "command": "python",
      "args": ["path/to/deepseek_server.py"],
      "env": {
        "DEEPSEEK_API_KEY": "your_api_key_here"
      }
    }
  }
}
```

#### 2. 环境变量设置
```bash
# Windows (PowerShell)
$env:DEEPSEEK_API_KEY="your_api_key_here"

# macOS/Linux
export DEEPSEEK_API_KEY="your_api_key_here"
```

## 核心功能详解

### 🔧 开发工具套件

#### 1. 功能开发工作流 (/feature-dev)

**🎯 功能概述**: 7阶段系统化功能开发流程

**🔄 工作流程**:
```
1. 需求分析 → 2. 代码探索 → 3. 架构设计 → 4. 实现规划 →
5. 编码实现 → 6. 测试验证 → 7. 代码审查
```

**💡 详细用法**:

##### 基本启动
```
启动功能开发工作流
```

##### 指定具体需求
```
我要为用户系统添加双因素认证功能
需要支持短信和邮箱验证
```

##### 带约束条件的开发
```
开发一个实时聊天功能，要求：
- 支持群聊和私聊
- 消息持久化存储
- 在线状态显示
- 低延迟 (<100ms)
```

**🎨 智能代理角色**:

- **代码探索者** (`code-explorer`): 深度分析现有代码库
- **代码架构师** (`code-architect`): 设计系统架构和实现方案
- **代码审查员** (`code-reviewer`): 质量保证和最佳实践检查

#### 2. 代码审查系统 (/code-review)

**🎯 功能概述**: 多维度自动化代码质量分析

**🔍 审查维度**:
- **代码质量**: 代码规范、最佳实践
- **性能分析**: 性能瓶颈、优化建议
- **安全检查**: 安全漏洞、敏感信息
- **可维护性**: 代码复杂度、可读性
- **测试覆盖**: 测试完整性、边界情况

**💡 使用方法**:

##### 审查当前变更
```
审查我最近的代码提交
```

##### 审查特定文件
```
审查 src/auth/user-service.ts 的代码质量
```

##### 专项审查
```
检查我的代码是否存在安全漏洞
分析这段代码的性能问题
```

**📊 审查报告格式**:
```
🔍 代码审查报告
├── 总体评分: 85/100 (良好)
├── 发现问题: 3个高优先级，5个中优先级
├── 性能建议: 2个优化点
├── 安全警告: 1个潜在风险
└── 改进建议: 详细修复方案
```

#### 3. 项目健康分析 (mcp-tools)

**🎯 功能概述**: 全方位项目健康状况评估

**📈 分析指标**:

##### 代码质量指标
- **圈复杂度**: 衡量代码逻辑复杂程度
- **函数长度**: 识别过长的函数
- **注释比率**: 代码文档化程度
- **重复代码**: 代码重复率分析

##### 项目活跃度指标
- **提交频率**: 项目开发活跃度
- **变更热点**: 频繁变更的文件
- **贡献者分布**: 团队参与度
- **分支管理**: 分支策略健康度

##### 测试覆盖指标
- **测试覆盖率**: 测试完整性
- **测试质量**: 测试有效性
- **边界测试**: 边界情况处理
- **集成测试**: 系统集成度

**💡 使用方法**:

##### 全面健康检查
```
分析我的项目健康状况
```

##### 专项分析
```
检查代码复杂度
分析测试覆盖率
识别技术债务
```

##### 比较分析
```
对比当前版本和上周版本的项目健康指标
```

### 🤝 协作工具套件

#### 1. Git自动化 (/commit)

**🎯 功能概述**: 智能化Git工作流程

**🔧 主要功能**:

##### 智能提交
```
创建提交                     # 基于变更自动生成提交信息
提交并推送                   # 提交后立即推送到远程
提交并创建PR                 # 完整的提交-推送-PR流程
```

**📝 提交信息智能生成**:
- 分析代码变更内容
- 识别变更类型(功能/修复/重构等)

```
清理已删除的远程分支         # 清理本地失效分支
```

**🎯 功能概述**: 专业化Pull Request审查

**🔍 审查流程**:
```
1. 代码变更分析 → 2. 质量检查 → 3. 性能评估 →
4. 安全扫描 → 5. 测试验证 → 6. 综合评价
```

**🤖 专业代理团队**:
