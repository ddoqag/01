# Project Dev 技能

## 概述

基于实战验证的"3-6-3 AI编程工作流程"的项目开发技能。通过三步六动作三坑点的专业方法论，帮助用户高效完成从需求分析到项目交付的全流程开发。

## 🎯 核心方法论

### 3步骤：完整开发流程
1. **需求拆解** - 清晰描述需求，补充详细文档，清空上下文
2. **生成实现** - 严格按需求生成，专注执行实现
3. **验收迭代** - 集中测试问题，批量修复完善

### 6动作：具体执行步骤
1. 需求清晰描述
2. 补充详细文档
3. 清空上下文环境
4. 严格按文档生成
5. 专注代码实现
6. 集中验收测试

### 3坑点：必须避免的陷阱
1. **避免零散修改** - 一次验收，批量修复
2. **分离功能与修复** - 专注当前迭代目标
3. **及时回退重生成** - 避免反复修改的低效

## 🎖️ 实战案例

### 配置切换工具开发
**背景**：面试中遇到的AI模型配置管理问题
**挑战**：手动切换配置效率低，API密钥安全风险高
**解决方案**：基于3-6-3方法论，200+次迭代开发
**成果**：
- CLI工具：查看、切换、验证配置
- Web GUI：可视化管理界面
- 安全设计：TOML配置 + source加载
- 跨平台：Windows/macOS/Linux支持

### 技术亮点
- 仅使用Node.js内置模块
- 最小化依赖，提高安全性
- 配置文件加密保护
- 热重载配置更新

## 🔧 最佳实践

### 需求描述技巧
1. **具体明确**：避免模糊描述，提供具体的功能要求
2. **技术约束**：明确技术栈、性能要求、安全要求
3. **验收标准**：定义清晰的成功标准和测试方法
4. **边界条件**：考虑异常情况、错误处理、用户体验

### 代码生成技巧
1. **详细文档**：提供完整的需求规格和技术设计
2. **严格提示**：使用"ultra strict"和"ultra detail"
3. **分步实现**：复杂项目分阶段生成和验证
4. **质量检查**：每个阶段都要进行质量验证

### 测试验收技巧
1. **全面覆盖**：功能、性能、安全、兼容性全面测试
2. **问题记录**：详细记录重现步骤和环境信息
3. **批量修复**：集中所有问题，一次性修复
4. **回归测试**：修复后进行全面的回归测试
